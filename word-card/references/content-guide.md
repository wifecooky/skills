# 5 元素内容生成规范

## 核心原则

一张词卡的职责：让用户看完记住**一个画面**，带走**一张词根网络**。

- 记住一件事记得牢，记十件事全忘光
- 人脑记图像的效率是文字的 6 倍
- 有情绪的东西不容易忘

## 5 个必备元素

### 1. 词根拆解 (Etymology)

追溯到拉丁/希腊语源，拆成可复用的构件。

**输出格式**: `prefix- (含义) + root (含义)`

**规范**:
- 必须给出原始语源语言（Latin / Greek / Old English）
- 每个构件用一个中文词翻译
- 如果是复合词根，逐段拆解
- 在 `{{WORD}}` 模板变量中，用 `<span class="prefix">` 和 `<span class="root">` 分别包裹

**示例**:
```
trans- (穿越, Latin) + scribere (书写, Latin)
re- (再次) + sili- (跳, Latin salire) + -ence (状态)
```

### 2. 核心汉字 (Hanzi)

一个汉字，提炼单词的**核心意象**。不是翻译，是意象提炼。

**规范**:
- 只选一个字
- 选意象最强烈、最具画面感的字
- 不能是直译（transcribe ≠ 抄），要是升华（transcribe = 渡）
- 这个字要能「镇住」整张卡片

**思考路径**: 这个词的动作/状态，如果只用一个画面表达，那个画面里最有力的一个字是什么？

### 3. 画面描写 (Scene)

40-60 字的具象场景。用户闭眼能「看见」。

**规范**:
- 必须有人物、动作、环境
- 用现在时，营造身临其境感
- 关键动作加 `<strong>` 强调
- 不要抽象描述（❌ 忠实地传递信息），要具象画面（✅ 一笔一画搬到另一页）

**示例**:
```
昏暗的修道院里，抄写员弓着背，将羊皮纸上的文字一笔一画搬到另一页。
<strong>不增不减，原样渡过。</strong>
```

### 4. 金句 (Epiphany)

一句英文 + 一句中文，中英**不是直译**关系。

**规范**:
- 英文句揭示词的本质动作/状态
- 中文句用不同的角度说同一件事
- 必须原创，不要引用名人名言
- 单词本身或核心意象用 `<span class="hi">` 高亮
- 要经得起单独抽出来的力量测试：脱离词卡上下文，这句话还有没有力量？

### 5. 真实例句 (Example)

一个英文例句 + 中文翻译。

**规范**:
- 必须是真实语境（新闻、文学、学术、日常），不能是编造的教科书句
- 单词本身用 `<em>` 标记
- 句子要展示单词在真实场景中的用法
- 中文翻译自然，不是逐词对应

## 词根家族星图数据 (Constellation Data)

每个词拆出的词根（通常 2 个），每个词根扩展 4 个同族词。

### JSON 格式

```json
{
  "nodes": [
    { "id": "hub", "type": "hub" },
    { "id": "transcend", "zh": "超越", "family": "trans", "pre": "trans", "suf": "cend" },
    { "id": "describe", "zh": "描述", "family": "scribe", "pre": "de", "suf": "scribe" }
  ],
  "links": [
    { "source": "hub", "target": "transcend", "family": "trans" },
    { "source": "hub", "target": "describe", "family": "scribe" }
  ]
}
```

### 节点字段

| 字段 | 说明 |
|------|------|
| `id` | 英文单词（小写） |
| `type` | 仅中心节点为 `"hub"` |
| `zh` | 中文释义（2-3 字） |
| `family` | 所属词根族名（如 `"trans"`, `"scribe"`） |
| `pre` | 标签前半段（浅色显示） |
| `suf` | 标签后半段（强调色显示） |

### 规范

- `hub` 节点必须在第一位
- 每个词根族**恰好 4 个**词（不多不少）
- 选择**常见且有画面感**的同族词，不要冷僻词
- `family` 名用词根本身（不带 `-`）
- `pre` + `suf` 拼接后 = 完整单词
- 所有 link 的 source 都是 `"hub"`

## REF_TAG

卡片顶部的小标签，格式为 `类别 / 子类`。

示例: `语言学 / 词源解构`、`物理 / 热力学`、`心理学 / 认知`

根据单词的学科领域选择，2-4 个字即可。
